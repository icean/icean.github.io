1. Hexo部署

软件安装

  • Node.js
  • Git
    • Windows建议下载Windows Portable版本的，下载地址，本文以Windows为例
    • Mac OS或者Linux可以在命令行下进行安装

注册GitHub

去GitHub注册一个帐号，已有请忽略

配置SSH keys

  • 运行git Bash，查看是否已经存在SSH Key，查看id_rsa.pub 或 id_dsa.pub是否存在，如果存在，跳过生成新的SSH key步骤

  • 生成新的SSH key

    -t 指定密钥类型，默认是 rsa ，可以省略。
    -C 设置注释文字，比如邮箱。
    -f 指定密钥文件存储文件名

    上述命令省略了-f参数，之后一直输入回车就可以，更多细节可以查看GitHub配置添加SSH

  • 添加SSH key到GitHub

    • 复制 ~/.ssh/id_rsa.pub内容
    • 登录GitHub账号，从右上角的设置（ Account Settings ）进入，然后点击菜单栏的 SSH key 进入页面添加 SSH key
    • 点击 Add SSH key 按钮添加一个 SSH key 。粘贴 ~/.ssh/id_rsa.pub内容到所对应的输入框中，SH key 代码的前后不要留有空格或者回车

  • SSH Key测试，git Bash中输入

    如果反馈如下

    输入yes，然后会看到，其中xxxx为GitHub用户名，说明SSH Key已经测试成功

2. Hexo订制 & 部署

目录结构

  • _config.yml 全局配置文件，网站的很多信息都在这里配置，诸如网站名称，副标题，描述，作者，语言，主题，部署等等参数。这个文件下面会做较为详细的介绍。
  • package.json hexo框架的参数和所依赖插件
  • scaffolds scaffolds是“脚手架、骨架”的意思，当新建一篇文章（hexo new ‘title’）的时候，hexo是根据这个目录下的文件进行构建的。基本不用关心。
  • source 这个目录很重要，新建的文章都是在保存在这个目录下的_posts ，\_posts 目录下是一个个 markdown 文件，会被编译成html文件，放到 public文件夹下
  • theme 网站主题目录，该目录下，每一个子目录就是一个主题

主题切换 && 订制

修改_config.yml文件theme选项，主题去GitHub等网站去搜索，根据个人需要，对主题文件可以进行修改

部署到GitHub

修改_config.yml文件，配置deploy选项

部署完成后，可以访问http://[GitHub用户名].github.io查看博客

评论添加

  • 注册多说账户
  • 在多说中设置域名等信息
  • 在Hexo主配置文件或者主题的主配置文件中，配置duoshuo_shortname字段，填入名称
  • 重新部署Hexo

统计添加

添加百度统计或者谷歌统计统计博客的一些访问量，PV信息

百度统计

  • 百度统计注册一个帐号，根据提示设置域名等信息
  • 一般在主题的主配置文件中，查找baidu_analytics类似选项，添加跟踪ID
  • 重新部署Hexo

谷歌统计

  • 谷歌统计可以使用Gmail账户登录
  • 一般在主题的主配置文件中，查找google_analytics类似选项，添加跟踪ID
  • 重新部署Hexo

独立域名绑定

  • 在source文件夹中，新建一个名为CNAME的文本文件，里面写入你要绑定的域名，例如blog.kompass.com
  • 在域名管理中，添加别名解析，将要绑定的独立域名解析到[GitHub用户名].github.io
  • 重新部署Hexo
