XX卡破解 - UID固化卡片数值修改

发表于 2017-03-01 | 更新于 2018-05-08 | 分类于 其他
使用ACR-122对XX卡进行破解。2008 年的时候，荷兰恩智浦（NXP）公司开发的 RFID 产品 Mifare Classic 就被破解了，详细请自行Google 或百度。RFID即为射频识别，NFC近场通信，实际上NFC可以理解为以RFID技术为基础的一种产品。

警告

本文仅为测试，请勿将此技术用于非法用途，后果自负！！！

1. 系列章节

  • XX卡破解 - 数值变化规律猜测
  • XX卡破解 - 相关字段查找

2. 相关探索

前文简介

前面的文章已经介绍的主要内容有

  • 可以将卡片的内容导出为dump，然后再导入到UID可写的卡片中
  • 猜测并验证了卡片数值变化的规律
  • 探索的卡片的相关字段

根据资料调研，在密钥正确的情况下，ACR-122u也可以对卡片的内容进行修改

探索操作

  • 对于UID固定的卡片，无法直接导入dump文件，即使dump文件对应的UID和卡片的UID相同
  • 在Windows7上已经安装好了ACR-122u的驱动，运行可以修改固化UID卡片数值的软件，但是软件无法读取卡片正确的UID，读取出的UID为00 00 00 00
  • 经过调研，确定了可以修改卡片数值的软件是运行在Windows XP系统上的

3. 环境搭建

所需要软件以及硬件如下

  • Windows XP虚拟机，安装.NET架构，否则写卡软件无法正确运行，下载地址
  • ACR-122u
  • UID固化卡片
  • M1工具包，下载地址

5. 总结

到此，XX卡破解系列结束，总结一下思路与经验

  • 确定数据是在线存储还是离线存储，所谓的在线存储，就是卡片只是用于身份验证，其他具体的数据，例如消费金额都存在服务器中，这是无法去做进一步工作的，离线存储就是数据存储的卡片，这样可以对卡片进行读取、查找规律，最后修改正确的数据
  • 观察卡片数值变化规律，使用winhex对比dump文件找出变化的字段
  • 探究数据变化规律，这个可能是最难的，但是也是最有意思的
  • 探究相关字段，修改哪些字段不影响卡片的正常使用，相关字段查找的思路可以为找两张正常使用，业务状态相同的卡片，然后导出dump文件，对比有哪些不同的字段，然后修改这些字段，修改后测试是否影响正常使用
  • UID固化卡片修改，因为软件是在Windows XP上使用的，之前测试都在Windows7上，所以兜了一个大圈子
